This is a simple addition example running on the [Embedded Proplet](/embedded-proplet). It takes two numbers as inputs and returns their sum, executing inside WAMR on an ESP32-S3 running Zephyr RTOS. The module is written in WebAssembly Text format because WAMR on Zephyr does not support WASI (all WASI interfaces are disabled in the build configuration). For architecture details, memory constraints, and configuration options, see the [Embedded Proplet](/embedded-proplet) documentation.

Before running this example you need:

- A Zephyr development environment — follow the [Zephyr Guide](/zephyr) for SDK, west, and toolchain installation
- The Propeller stack running (Manager + MQTT broker) — follow the [Getting Started](/getting-started) guide
- `wat2wasm` from the [WebAssembly Binary Toolkit](https://github.com/WebAssembly/wabt)

Initialize the WAMR submodule:

```bash
cd propeller/embed-proplet
git submodule update --init --recursive
```

Your output should look like this:

```bash
Submodule 'modules/wamr/wasm-micro-runtime' (https://github.com/bytecodealliance/wasm-micro-runtime.git) registered for path 'modules/wamr/wasm-micro-runtime'
Cloning into '/path/to/propeller/embed-proplet/modules/wamr/wasm-micro-runtime'...
Submodule path 'modules/wamr/wasm-micro-runtime': checked out 'abc1234...'
```

Configure your device identity and WiFi credentials in `src/main.c`:

```c
#define WIFI_SSID     "your_wifi_ssid"
#define WIFI_PSK      "your_wifi_password"
#define PROPLET_ID    "proplet-esp32-001"
#define DOMAIN_ID     "your_domain_id"
#define CHANNEL_ID    "your_channel_id"
```

Set the MQTT broker IP address in `src/mqtt_client.c` to the host running your Propeller stack:

```c
#define MQTT_BROKER_HOSTNAME "10.42.0.1" /* Replace with your broker's IP */
```

Build and flash the firmware:

```bash
west build -b esp32s3_devkitc/esp32s3/procpu -p auto .
west flash
west espressif monitor
```

## Create Task

To create a task for this example, we need to build the example to wasm and then create a task with the wasm file.

```bash
cd propeller
make addition-wat
```

Now we can create a task with the wasm file:

```bash
curl -X POST "http://localhost:7070/tasks" \
-H "Content-Type: application/json" \
-d '{"name": "main", "inputs": [10, 20]}'
```

Your output should look like this:

```json
{
```


## Start Task

Now we can start the task:

```bash
curl -X POST "http://localhost:7070/tasks/b8fabc60-1156-4883-a2cd-4b70380ea5df/start"
```

Your output should look like this:

```json
{ "started": true }
```
